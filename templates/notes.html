Lesson 12: Web Server Allowing Comments

Terms:

Network - Is a group of entities that can communicate although they are not directly connected. For example a triangle, not all the points are connected but they can still communicate from one to the other by forwarding the messages back

Latency - the time it take to communicate from the source to its destiantion. This is one type of way to measure a network.

Bandwith - Is the amount of information that is being emiited per unit of time, which can often be measure in Mbps (megabits per second). This is the second way to measure a network.

Protocol - What is used to make a network work. Its a set of rules that allow to entities to talk to one another.

The protocol gives the client (Web Browser) a way to talk with the server. This protocol is called HTTP (Hyper Text Transfer Protocol).
The Message GET. The client sends a GET <object> , which gets the server recieves to attain the file requested and send the response back of the requestet content of <object>.

Server

Servers - Servers will deliver either a static or a dynamic response dependinh on what is requested. Static responses are pre-written files like images that are requested.
Dynamic responses are built in ready to respond by programs running on the server.

HyperText Transfer Protocol (HTTP) - is a protocol used on the world wide web. The two most important methods we learned are GET and POST in HTTTP.

GET - A GET is used for fetching documents. The client sends a GET <object> , which gets the server recieves to attain the file requested and send the response back of the requestet content of <object>.
Example is GET /path HTTP/1.1 . The Server then reads this information and acts upon sending a response back. These parameters appear are in the URL.

POST - A POST requests change data on the Server. POST requests on the other hand put them into the HTTP headers and parameters are in the body. There is no maximum limit but this can be configured.
POST is most often used for sending data to the server. There are others, but these are the ones we'll use in this course.

Validation of User Input
Validation - means verifying on the server side that what we received is what we expected.
In the case of users submitting data via html forms, a defined set of input would always be expected.
The server should not process just any data sent because a user can maliciously input large amounts of data to slow down or corrupt the server.
A User should not be able to input javascript, because this will cause the server to have abnormal behaviore creating responses that dont make sense.

There are many ways to ensure the server receives only valid data like using a drop-down menu items or buttons would help from allowing the user to have invalid entries.
The most effective way is to verify all received data meets compliance with expected data formats.

Within Jinja2 we has an autoescape feature which is a secure procedure when building websites to avoid accidental oversight of security openings like entering HTML code in an Input box on a website.

User Experience - Having validations allows the user not to make a mistake allowing them to get exactly what they want, rather than expecting something that was not expected. Validation help the user to enjoy what the developer has to offer as to what frustration the user may go through. For example if I am playing a game and none of my inputs are correct nor validated. It will cause me to get frustrated with the game due to the game not working properly due to my inputs being not validated. If I recieved validation it will let me know where I went wrong allowing me to fix my mistakes and to enjoy the game the devloper created.
